SIM Monitor

面向多张实体 SIM 卡的保号与余额管理工具。它把卡片信息、运营商规则、活跃记录和到期提醒组织在一起，让备用号码不再悄悄失效。

这是一个偏工具型的原生 Android 应用：不读取短信、不抓取通话记录、不自动查询运营商账户，而是用本地数据库和规则引擎帮用户清楚管理“什么时候需要做一次保号动作”。

Core Features

把 SIM 卡保号变成可见的日程

01

多卡集中管理

把海外卡、备用号、实体 SIM 和低频号码放进同一个视图，记录运营商、余额、币种、激活时间与备注。

02

保号规则巡检

根据 Giffgaff、CTExcel 或通用 180 天规则计算下一次保号动作，自动判断健康、关注、风险和过期状态。

03

本地提醒

后台每日检查卡片状态，在临近失效前通过 Android 本地通知提醒充值、发短信、拨打电话或使用流量。

04

操作历史可追踪

充值、短信、通话、流量、套餐购买和付费保号服务都能留下记录，后续查账和复盘更清楚。

Risk Model

清楚知道哪张卡需要立刻处理

SIM Monitor 会基于绑定规则、最近活跃时间和用户记录生成状态，让多卡管理不再依赖记忆或零散备忘录。

HEALTHY 状态健康，距离下一次保号动作仍有较长时间。
ATTENTION 进入关注期，建议提前准备下一次保号动作。
RISK 临近失效，需要尽快执行短信、通话或充值等动作。
EXPIRED 规则判断已过期，需要立即核实实体卡状态。

How It Works

从录入到提醒，只保留必要步骤

  1. 录入 SIM 卡基础信息和余额
  2. 绑定运营商或自定义保号规则
  3. 记录充值、短信、通话、流量等动作
  4. 每日后台巡检并生成提醒

Build

Kotlin + Jetpack Compose 原生实现

项目使用 Room 保存 SIM 卡、规则、使用记录和提醒记录；DataStore 保存本地设置；WorkManager 执行每日巡检；Compose 和 Material 3 构建界面。
